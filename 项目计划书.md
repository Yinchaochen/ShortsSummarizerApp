# Shorts Summarizer — 项目计划书

作成日期：2026-04-06

---

## 一、产品概述

Shorts Summarizer 是一款 App + 网站，帮助用户快速理解短视频内容。
用户粘贴一个短视频链接，系统自动下载、分析并输出详细总结。
核心服务对象：不同语言/文化背景的用户，跨文化情侣、异地家庭、内容研究者。

---

## 二、核心功能（Phase 1）

- 用户注册/登录（邮箱 + Google + Apple）
- 选择总结语言（中文、英文、日文等）
- 输入短视频链接（Phase 1 支持 TikTok）
- 实时进度条显示处理状态
- 输出详细视频总结
- 免费用户限额：每账号 3 次免费试用
- 历史记录查看

## 四、技术架构

```
Expo App（iOS / Android / Web 三端同一套代码）
        ↓
FastAPI 后端（Python）
        ↓
Celery + Redis 异步任务队列
        ↓
平台下载层（yt-dlp）      Gemini API（视频分析）
        ↓
Supabase（认证 + PostgreSQL 数据库 + Storage）
```

---

## 五、技术选型

| 模块 | 技术 | 理由 |
|------|------|------|
| 前端 / App | Expo (React Native + TypeScript) | 一套代码跑 iOS / Android / Web |
| 后端 | FastAPI (Python) | 与现有 Python 代码兼容，异步支持好 |
| 任务队列 | Celery + Redis | 处理高并发视频分析任务 |
| 认证 + 数据库 | Supabase | 开箱即用，免费额度覆盖早期用户 |
| 视频下载 | yt-dlp | 支持 TikTok / Instagram / YouTube |
| AI 分析 | Gemini 2.5 Flash | 原生视频理解，多语言支持 |
| 支付（Web） | Stripe | 主流支付方案 |
| 支付（App） | RevenueCat | iOS / Android 内购管理 |
| Web 部署 | Vercel | 免费，自动 CI/CD |
| 后端部署 | Railway | Python 友好，支持 Celery Worker |
| Redis | Upstash | 托管 Redis，免费起步 |

## 六、平台支持策略

Phase 1 仅支持 **TikTok**（yt-dlp 无需登录，风险最低）。

架构中已预留平台抽象层，后续添加平台只需新增对应 handler：

```
services/platforms/
    ├── base.py         # 抽象基类
    ├── tiktok.py       # Phase 1 上线
    ├── instagram.py    # Phase 2（服务账号模式）
    └── youtube.py      # Phase 2
```

## 七、数据库设计（Supabase）

```sql
-- 视频总结记录
summaries (
    id          UUID PRIMARY KEY,
    user_id     UUID REFERENCES auth.users,
    url         TEXT,
    platform    TEXT,        -- 'tiktok' | 'instagram' | 'youtube'
    language    TEXT,
    result      TEXT,
    created_at  TIMESTAMP
)

-- 用户使用限额
usage (
    user_id     UUID PRIMARY KEY,
    free_count  INT DEFAULT 0,   -- 已使用免费次数
    reset_at    TIMESTAMP        -- 预留：限额重置时间
)
```

## 十一、关键风险与应对

| 风险 | 应对方案 |
|------|------|
| TikTok 封锁 yt-dlp | 预留 RapidAPI 等第三方解析 API 接口 |
| Gemini API 成本失控 | 缓存相同 URL + 每用户限额 |
| App Store 审核拒绝 | 避免在描述中提及"下载视频" |
| Instagram 封号 | Phase 1 不支持 Instagram，Phase 2 再处理 |
| 高并发压力 | Celery Worker 横向扩展，Redis 队列缓冲 |
